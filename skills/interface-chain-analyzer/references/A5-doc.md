# A5 API 文档生成

## 输出目标

根据代码 diff、接口链路和现有上下文补齐接口文档或接口变更说明，形成统一、可复用、可联调且便于测试执行的 API 说明。

## 固定输出结构

1. 接口概述
2. 请求定义
3. 处理逻辑摘要
4. 响应定义
5. 异常场景
6. 测试与联调提示
7. 注意事项

## 分析要求

1. 优先明确本次变更影响的请求路径、方法、参数、返回信息和兼容性影响。
2. 对链路中可以确定的处理逻辑做摘要，不扩写无依据内容。
3. `测试与联调提示` 至少包含：关键前置数据、建议优先校验字段、关键日志/状态观测点。
4. 若存在 `relatedInterfaces`，应优先把其作为受影响接口文档或接口变更说明的证据来源。
5. 若能识别幂等、状态流转、外部依赖或异步结果，也应在“测试与联调提示”中说明。
6. 若 diff 无法稳定定位明确接口，应输出“受影响 API/字段待确认”的说明，而不是编造接口细节。
7. 对无法确认的字段标记为待补充。
