# A5 API 文档生成

## 输出目标

根据接口链路和现有上下文补齐接口文档，形成统一、可复用、可联调且便于测试执行的 API 说明。

## 固定输出结构

1. 接口概述
2. 请求定义
3. 处理逻辑摘要
4. 响应定义
5. 异常场景
6. 测试与联调提示
7. 注意事项

## 分析要求

1. 明确请求路径、方法、参数和返回信息。
2. 对链路中可以确定的处理逻辑做摘要，不扩写无依据内容。
3. `测试与联调提示` 至少包含：关键前置数据、建议优先校验字段、关键日志/状态观测点。
4. 若能识别幂等、状态流转、外部依赖或异步结果，也应在“测试与联调提示”中说明。
5. 对无法确认的字段标记为待补充。
