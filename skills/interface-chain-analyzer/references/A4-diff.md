# A4 Diff 关联接口用例生成

## 输出目标

根据 git diff、受影响代码和接口链路，识别关联接口并生成增量测试与回归测试用例。

## 固定输出结构

1. 变更摘要
2. 关联接口识别
3. 增量测试用例
4. 回归建议
5. 待确认项

## 分析要求

1. 优先识别本次 diff 直接影响的 Controller、Service、Repository、DTO、异常处理、状态流转和返回结构。
2. 必须区分直接影响接口和间接受影响接口，并说明关联依据。
3. 变更摘要必须指出本次变更属于哪类影响：请求参数、响应字段、状态机、异常处理、依赖调用、持久化、鉴权、幂等、日志/审计。
4. 增量测试用例必须按“新增验证点 / 需要重点回归的旧能力”组织，而不是重写一份全量接口测试清单。
5. 每条用例都要说明触发原因，原因必须来自 diff、受影响方法或关联接口证据。
6. 回归建议必须区分：必回归、建议回归、暂不回归。
7. 对无法从 diff 直接确认的部分标记为待确认，禁止编造。
8. 若存在工具输出的 relatedInterfaces，优先以该结构化结果作为关联接口证据来源。

## 输出约束

1. “关联接口识别”至少包含：接口路径、请求方法、处理器/控制器、关联类型、关联依据。
2. “增量测试用例”至少包含：用例编号、关联接口、变更点、触发原因、优先级、测试步骤、预期结果。
3. 若 diff 无法定位到明确接口，必须输出“未识别到明确接口，仅能给出代码层回归建议”。
