# interface-chain-analyzer

## 1. 目标

本 Skill 用于消费接口链路数据，并在固定结构下生成四类分析结果，避免模型自由发挥造成的格式漂移和覆盖不稳定。

## 2. 意图路由规则

当用户请求符合以下意图时，必须映射到指定节点：

| 节点 | 意图 | 典型关键词 |
| --- | --- | --- |
| A1 | 业务逻辑梳理 | 业务流程、逻辑梳理、链路解读、流程图 |
| A2 | Bug 风险分析 | 风险分析、异常点、Bug、隐患、边界问题 |
| A3 | 测试用例生成 | 测试用例、测试点、覆盖设计、场景设计 |
| A4 | Diff 关联接口用例生成 | git diff、代码变更、变更影响、增量测试、回归用例、关联接口 |
| A5 | API 文档生成 | API 文档、接口说明、请求响应、文档补齐 |

## 3. 强制执行顺序

1. 先读取本文件，确定用户意图与节点映射。
2. 再读取目标节点对应的 reference 文件。
3. 严格按照 reference 文件中的输出结构组织结果。
4. 信息不足时必须输出“待确认项”或“无法判断”，禁止编造。

## 4. 共通分析原则

1. 优先基于链路数据和明确证据下结论。
2. 只有在必要时才补充方法源码查询。
3. 结论必须尽量给出依据、触发条件或影响范围。
4. 输出面向测试、开发和质量协作，要求简洁、可执行、可复用。
5. 如果输入数据缺少关键字段，应在结果中明确提示缺口。

## 5. 输出约束

1. 标题和章节顺序必须稳定。
2. 表格列名必须稳定。
3. 相同类型任务应保证输出结构一致。
4. 不允许输出与当前链路无关的泛化废话。
