# 仓库开发约束

本仓库后续开发默认遵循 [docs/开发技术栈与开发规范_V1.0.md](docs/%E5%BC%80%E5%8F%91%E6%8A%80%E6%9C%AF%E6%A0%88%E4%B8%8E%E5%BC%80%E5%8F%91%E8%A7%84%E8%8C%83_V1.0.md)。如与局部实现冲突，以“新增代码遵循规范、存量代码逐步治理”为原则推进。

## 强制技术基线

1. 后端使用 Java 8 和 Spring Boot 2.7.6。
2. 前端使用现有项目既定的 Vue 主版本，不允许混用 Vue 2 和 Vue 3。
3. 数据层使用 MySQL、Redis、Flyway、MyBatis-Plus。
4. 不得私自升级 JDK、Vue、MySQL、Redis 主版本。

## 开发约束

1. Controller 只做接参、校验和响应封装，不写复杂业务。
2. Service 承载业务逻辑和事务边界，不直接拼接 SQL。
3. DAO/Mapper 只负责数据访问。
4. 新增代码必须兼容 JDK 1.8，禁止使用高版本 Java 语法。
5. API 调用逻辑集中管理，避免重复实现。
6. 数据库变更必须通过脚本或 Flyway 管理，禁止直接手改生产库。
7. Redis key 按“业务前缀:模块:主键”规范命名，并设置合理 TTL。
8. 敏感信息不得写入仓库或日志明文。

## 交付要求

1. 新增功能需补充必要注释，复杂逻辑要说明意图。
2. 提交前应完成格式化、静态检查和必要测试。
3. 涉及数据库、缓存结构或公共接口变更时，必须说明兼容策略和回滚方案。
4. 如果需要偏离本规范，必须先说明原因并获得确认。
